# Prospex CRE

Prospex CRE is a Commercial Development Intelligence Platform for exploring census tract level commercial real estate development suitability. The app displays tract scores generated by a Spatially Weighted Random Forest model and surfaces SHAP-style drivers so users can understand why a tract was selected.

## Setup

Install dependencies:

```bash
npm install
```

Run the local development server:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

## Data File Location

Place the GeoJSON file here:

```text
public/data/prospex_cre_v1_tract_scores.geojson
```

The application loads the file directly from the public folder. No backend is required for V1.

## Expected GeoJSON Fields

The app expects one feature per census tract. It is defensive about missing or renamed fields, but the best experience comes from including:

- `GEOID`
- `county_name`
- `NAMELSAD` or another tract name field
- `prospex_score_100`
- `gis_suitability_score_100`
- `recent_construction_rate`
- `population_density`
- `housing_density`
- `median_income`
- `median_home_value`
- `median_rent`
- `dist_major_road_km`
- `poi_density`
- `top_positive_driver_1`
- `top_positive_driver_2`
- `top_positive_driver_3`
- `top_negative_driver_1`
- `top_negative_driver_2`
- `top_negative_driver_3`

If SHAP fields are missing, the tract info card and top tract list display fallback messaging instead of crashing. SHAP field parsing is centralized in `src/utils/tractData.ts`.

## Local Workflow

1. Add `prospex_cre_v1_tract_scores.geojson` to `public/data`.
2. Run `npm install`.
3. Run `npm run dev`.
4. Open the Vite URL shown in the terminal.
5. Hover tracts on the map to inspect Prospex score, baseline GIS score, recent construction signal, and top model drivers.
6. Use the county selector to update the top 10 tract list for All Counties, Miami Dade, Broward, or Palm Beach.
